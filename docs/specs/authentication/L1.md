# AUTH — Authentication

High-level (L1) requirements for the **Authentication** capability. Detailed L2 requirements live in [L2.md](./L2.md).

---

## L1-AUTH-001: User Authentication & Identity
*(formerly L1-001)*

The system must let a person register an account and sign in with email and password, must
issue a signed access token establishing their identity, and must expose the currently
authenticated user's profile (including derived initials) to authorized callers. Registration
also provisions the registrant's root account — a personal `Workspace` (the "Account") with a
Lead `Membership` — unless the registrant presents a valid invitation, in which case they join
the inviting account instead.
