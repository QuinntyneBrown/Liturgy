# WORK — Work Management

High-level (L1) requirements for the **Work Management** capability. Detailed L2 requirements live in [L2.md](./L2.md).

## L1-WORK-001: Develop Kanban Board
*(formerly L1-007)*

The system must present the Develop phase as a Kanban board with Backlog, In-the-5R-loop,
Review, and Done columns; must let members create, assign, and move cards (the "tickets");
and must display each card's 5R progress. The Done column must be locked so that no card can
enter it until its 5R loop is complete. A card (**Ticket = `Card`**) carries an optional
`Description` (set at creation) and optional story `Points`, and a `CardStatus {Open, Closed,
Cancelled}`. Members must be able to point a card (story points), to close or cancel it
(terminal states that remove it from the active board), to reopen a closed or cancelled card,
and to delete a card outright. `Close` and `Cancel` are allowed from any 5R state and are independent of the Done
gate — they end a card's active life without asserting a complete loop.

## L1-WORK-002: 5R Co-Creation Loop
*(formerly L1-008)*

The system must run the five-movement 5R loop on every work item, must require the movements to
be logged strictly in order, must capture the reflective content of each movement, and must not
permit a card to be marked Done until all five movements are logged.
