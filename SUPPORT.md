# Support

## Getting help

Before asking for help:

1. Review the [README](README.md), the [architecture documentation](docs/ARCHITECTURE.md), and the [implementation plan](docs/implementation-plan.md).
2. Search [existing issues](https://github.com/QuinntyneBrown/Liturgy/issues) for the same problem.
3. Confirm the problem still occurs on the latest `main` branch with supported .NET, Node.js, and SQL Server versions.

For reproducible defects, open a [bug report](https://github.com/QuinntyneBrown/Liturgy/issues/new?template=bug_report.yml). For scoped improvements, open a [feature request](https://github.com/QuinntyneBrown/Liturgy/issues/new?template=feature_request.yml).

Include the expected and actual behaviour, reproduction steps, relevant environment details (OS, .NET SDK, Node version, SQL Server edition), and sanitized logs or screenshots. Never post credentials, signing keys, connection strings, or security vulnerabilities.

## Security and conduct

- Report vulnerabilities privately according to [SECURITY.md](SECURITY.md).
- Report conduct concerns through the private channel described in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

Support is provided by project maintainers and contributors on a best-effort basis. This repository does not include a service-level agreement or production operational support.
